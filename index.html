Задачи В течение семестра необходимо сдать 4 задачи: адаптивная верстка, клиентские скрипты - фотогалерея, серверные скрипты - форум, модуль для Apache (низкий уровень) или веб сервис (прикладное программирование). Сданные задачи отслеживать на гуглдоках. Верстка. Позиционируем разрабатываемый сайт как "old-school" профессиональное портфолио, ссылку на который не стыдно разместить на визитке при встрече с потенциальным работодателем. Срок сдачи - до 28.10.2014. При показе на широком экране (например, от 768px) дизайн предполагается стандартным. Опишем его сверху вниз: Верхняя часть - логотип. Далее - горизонтальное меню на основе текста. Закладочки, скругленные уголки, надписи под углом (типа стикеров на кнопке), градиенты, подсветки и прочие анимации привествуются. Далее - с левого бока навигация с картинками. Если студент специализируется в вебе, то сылки могут быть на сделанные им ранее сайты, картинки - это минискриншоты главных страниц. Если в ACM - фото с чемпионатов, если в робототехнике - то фото его роботов и т. д. Далее - с правого бока основной контент с кратикой и текстом. В самом низу футер с копирайтами, ссылками, контактами и т. п. При уменьшении ширины экрана (вплоть до 768 px) картинки масштабируются, т. е. тоже пропорционально уменьшаются. При дальнейшем уменьшении ширины экрана (например, от 550px до 768px) навигация "уплывает" под основной контент. Верхнее меню не занимает несколько строк, уменьшаются отступы между пунктами меню. При дальнейшем уменьшении ширины экрана (например, менее 550px) верхнее меню, которое было горизонтальным, перестраивается в вертикальное и занимает несколько (сколько ссылок) строк. Далее можно убрать картинки из основного контента. При повороте планшета на 90 градусов (смене вертикали и горизонтали) макет должен перестраиваться соответствующим образом. Фэйк или бредогенерация запрещены, контент должен быть содержательным. Сайт должен выглядеть почти одинаково во всех популярных современных браузерах (в частности, IE от 9 версии; Chrome от 31 версии; Firefox от 27 версии). Готовые CSS-фреймворкаи типа Bootstrap использовать запрещено. Если для анимации очень (!) нужна библиотека jQuery, ее использование можно согласовать с преподавателем. Однако все, что можно сделать без скриптов на чистом CSS, должно быть сделано без скриптов. За образец можно взять сайт, созданный в поддержку книги Бена Фрейна (Ben Frain) HTML5 и CSS3: Разработка сайтов для любых браузеров и устройств. Фотогалерея. Фотографии располагаются на сервере. Срок сдачи - до 18.11.2014. При первой загрузке отображается лента миниатюр. Если не все миниатюры вмещаются на экран, то по бокам отображаются кнопки-стрелки "далее"/"назад" для перемотки ленты. По щелчку на кнопке-стрелке лента сдвигается вправо/влево соответственно. Сдвиг может быть либо циклический, либо до последней/первой миниатюры, после чего щелчки по кнопке-стрелке далее/назад не сдвигают ленту далее/назад соответственно. При щелчке на мниатюре разворачивается темный полупрозрачный div, в котором отображаются фотографии. Миниатюры становятся некликабельными, так как лежат под слоем div'а. Кликабельность кнопкок-стрелок "далее"/"назад" на усмотрение студента, т. е. спсособ перемещения по фотографиям вперед/назад во время просмотра не оговаривается, но он должен быть. Отображается кнопка "закрыть просмотр", при щелчке на кнопке "закрыть просмотр" темный полупрозрачный div исчезает, мниатюра становятся кликабельными. Пока фотография не загрузилась с сервера отображается прогресс-бар, который создает видимость работы, чтобы пользователь понимал, что процесс загрузки идет. Например, могут отображаться крутящиеся кругляши, или крутящиеся песочные часы. Допустимо использование библиотек общего назначения, например, jQuery(). Готовые фотогалереи использовать нельзя. Привествуется реализация "плюшек", как то анимаций, загрузка фотографий на сервер (это уже серверный сценарий). Форум. Использовать готовые форумные движки нельзя; использование фреймворков общего назначения согласовывается с преподавателем в индивиуальном порядке (в частности, Bootstrap и Django разрешены). Дизайн красивый должен быть и радовать глаз, можно без изысков. Неописаные детали постановки - авторское право студента. Срок сдачи - до 16.12.2014. Студент выбирает серверную платформу по своему усмотрению. Это может быть, например, ОС Unix, веб сервер Apache с модулем обработки серверных сценариев на PHP (можно использовать другой язык, например, Python, Ruby или др.), СУБД MySQL. Можно использовать ОС типа Windows, ASP.NET, язык C#. Студент выбирет тематику форума (непротиворечащую действующему законодательству РФ и нормам морали), например, форум о туризме. Весь форум состоит из разделов (конные походы, сплавы по рекам, посещение пещер и т. д.). В каждом разделе есть темы (например, весенний сплав по Чусовой для семей с детьми [в разделе сплавы по рекам], сплав по р. Мылый Нарын [в разделе сплавы по рекам], на конях по Башкирии [в разделе конные походы]). Раздел может содержать несколько тем, каждая тема принадлеит только одному разделу. В каждой теме пользователи могут оставлять сообщения; сообщения состоит из текста сообщения, даты написания, автора. Данные по разделам, категориям и сообщениям, по позльзователям и их правам хранятся в базе данных. Структуру БД студент продумавает сам. Все пользователи форума разделены на категории: админы, авторы и гости. Админ может создавать разделы и категории, удалять разделы и категории со всеми сообщениями, удалять сообщения пользователей. Админ может удалять пользователей, при этом соообщения удаленных пользователей могут либо быть удалены, либо помечены как "сообщения удаленного пользователя" (на выбор студента). Авторы могу создавать темы, писать сообщения, удалять свои сообщения. Авторы могут удалить свой профиль. Гости могут читать сообщения. Предусмотрена регистрация пользователя на форуме. Необходимо вводить логин, пароль, адрес электронной почты. Для предотвращения регистрации роботов предусмотрена каптча и отсылка сообщения с уникальной ссылкой на почту для проверки валидности почтового адреса. Зарегистрированный пользователь получает права автора. Регистрация второго админа не предусмотрена. Предусмотрена авторизация пользователя на форуме. Необходимо вводить логин, пароль. Для предотвращения взлома пароля роботами путем простого перебора, предусмотрена каптча и блокировка по IP адресу. Авторизация предполагает запуск сеанса (SESSION). Неавторизированные потзователи имеют права гостя. Страницы для чтения и подачи сообщения предусматривают навигационные поля, поля сеанса, основной блок для вывода текста сообщений, авторства и даты сообщений, поля для ввода текста сообщений. При первом в рамках текущего сеанса обращении сервером высылаются затребованные категории раздела или сообщения выбранной категории. Необходимо продемонстрировать знание AJAX. Какой именно модуль будет использовать AJAX, студент выбирает самостоятельно. Например, это может быть модуль "Кто на сайте". Запрос клиента отправляются на сервер, где он обрабатывается. С сервера обратно высылается запрошенный блок данных (формат JSON или XML), которые парсятся клиентом и отображаются на странице без ее перезагрузки. Использование jQuery приветствуется. Привествуется реализация "плюшек", как то, защита от флуда (ограничения числа сообщений от автора в минуту), загрузка фотографий, фильтр мата, постраничный вывод соощений в больших категориях (по 25, 50, 100 шт.), создание модулей типа "Кто на сайте", "Самые комментируемые темы", "Поиск по ключевым словам" и т. д. Два варианта. Сделать модуль для веб сервера Apache. Назначение модуля согласовывается с каждым студентом индивидуально. Например, модуль может работать как простой фильтр и вырезать http-заголовки Range из запроса. Второй вариант - разобраться с WSDL и сделать небольшой иллюстрационный пример, показывающий суть WSDL. К концу семестра формулировка 4-й задачи будет уточнена.
